Lesson 0

Getting Your Bearings: What is a Command Line?

+
Goal: Understand what the command line is and why we use it.

What is the Command Line?

You normally use your computer by clicking on icons, menus, and buttons. That's called a graphical interface.

The command line (also called the terminal) is a different way to talk to your computer. Instead of clicking, you type instructions. It looks like a black window with text in it.

Think of it like this:

  • Graphical interface = pointing at a menu item in a restaurant
  • Command line = writing your order on a note to the kitchen

Both get the job done! The command line just gives you more control.

Why Do We Need It?

Claude Code runs inside the command line. It's like a chat window where you talk to the AI. You won't need to memorize a bunch of commands — just enough to get Claude started, and then Claude does the hard work!

Don't worry! The command line can look intimidating at first, but you'll only need a handful of simple commands. And once Claude is running, you just type in plain English.
Lesson 1

Opening the Command Line on Windows

+
Goal: Open a terminal (command line) window on your Windows computer.

Windows has a few different command line programs. We'll use PowerShell — it comes pre-installed on every modern Windows computer.

Steps to Open PowerShell

1

Press the Windows key on your keyboard (it has the Windows logo on it, usually near the bottom-left).

2

Type powershell — you'll see it appear in the search results.

3

Click on "Windows PowerShell" (the blue icon). Don't pick "Run as administrator" for now.

4

A blue/black window will open with some text. You'll see something like:

PS C:\Users\YourName>

This is your command line! The blinking cursor is where you type.

Tip: You can also right-click the Start button (Windows icon at the bottom-left) and choose "Terminal" or "Windows PowerShell" from the menu.

Try It Out!

Type this and press Enter:

echo "Hello, I'm using the command line!"

You should see your message printed back to you. Congrats — you just ran your first command!

Clear the screen cls
Lesson 2

Installing Claude Code

+
Goal: Install Claude Code on your Windows computer using the official installer.

Installing Claude Code on Windows is easy — it's a single command that downloads and sets everything up for you. No extra software needed!

Steps

1

Open PowerShell (press Windows key, type powershell, click it).

2

Copy and paste this command, then press Enter:

irm https://claude.ai/install.ps1 | iex

What this means:

  • irm = "Invoke-RestMethod" — downloads a file from the internet
  • https://claude.ai/install.ps1 = the official Claude Code installer script
  • | iex = runs the downloaded script to install Claude Code
3

Wait for it to finish. You'll see some text as it installs — that's normal!

4

Close your PowerShell window and open a new one (this is important so PowerShell knows about the new installation).

5

Verify it worked by typing:

claude --version

You should see a version number. If you do — Claude Code is installed!

Alternative: Install with WinGet

If the above method doesn't work, you can also install using WinGet (Windows' built-in package manager):

winget install Anthropic.ClaudeCode
Auto-updates: The native installer (the first method) automatically keeps Claude Code up to date in the background. No need to manually update!
If you see an error: Make sure you closed and reopened PowerShell after installing. If it still doesn't work, try restarting your computer and running the install command again.
Lesson 3

Setting Up Your Anthropic Account & API Key

+
Goal: Create an Anthropic account and log in so Claude Code can work.

Claude Code needs to connect to Anthropic's servers (the company that makes Claude). You'll need an account.

Option A: Log In with Your Browser (Easiest)

1

In PowerShell, type:

claude
2

The first time you run Claude, it will ask you to log in. Choose the option to log in with your browser.

3

Your browser will open. Sign up for an Anthropic account (or log in if you have one).

4

Approve the connection. The terminal will confirm you're logged in.

Option B: Use an API Key

If someone gave you an API key, or you want to use one from the Anthropic Console:

1

Go to console.anthropic.com in your browser and create an account.

2

Navigate to API Keys and create a new key. Copy it.

3

In PowerShell, set your key:

$env:ANTHROPIC_API_KEY = "your-key-here"

Replace your-key-here with the key you copied.

Important: Your API key is like a password — don't share it with anyone or post it online!
Lesson 4

Starting Claude Code for the First Time

+
Goal: Launch Claude Code and have your first conversation with it.

Create a Project Folder

Before starting Claude, let's create a folder for your first project. Claude works best when you start it inside a folder for the thing you want to build.

1

In PowerShell, create a new folder and go into it:

mkdir my-first-project
cd my-first-project

mkdir = "make directory" (folder). cd = "change directory" (go into it).

Launch Claude

2

Type:

claude

You'll see Claude's welcome message and a prompt where you can start typing.

Say Hello!

3

Try typing something like:

Hi Claude! Can you create a simple HTML file that says "Hello World" with a nice design?
4

Claude will write the code and ask your permission to create a file. Type y (yes) and press Enter to approve.

5

Open the file! You can find it in your my-first-project folder using File Explorer.

To quickly open the folder, type this in PowerShell:

explorer .

Then double-click the HTML file to open it in your browser.

Congrats! You just used AI to create your first web page! It's that easy.
Lesson 5

Understanding How Claude Code Works

+
Goal: Learn the key concepts of working with Claude Code.

The Conversation Flow

Working with Claude Code is like having a conversation:

  1. You ask for something in plain English
  2. Claude thinks about what to do
  3. Claude asks permission before making changes (creating files, running commands)
  4. You approve or deny each action
  5. Repeat until you're happy with the result

Key Things to Know

Approving Actions

Claude will ask before creating/editing files or running commands. You'll see a prompt asking you to approve:

  • y = Yes, do it
  • n = No, don't do that
Sending Messages

Press Enter to send your message to Claude. If you need multiple lines, press Shift + Enter for a new line without sending.

Stopping Claude

If Claude is doing something and you want it to stop, press Escape. To exit Claude completely, type /exit or press Ctrl + C twice.

Starting Fresh

Type /clear to clear the conversation and start over. This is helpful when switching to a new task.

Helpful Slash Commands

Command What It Does
/help Shows available commands and tips
/clear Clears conversation history, starts fresh
/exit Quits Claude Code
/compact Summarizes the conversation to save memory
Lesson 6

Writing Good Prompts (Talking to Claude Effectively)

+
Goal: Learn how to ask Claude for things in a way that gets the best results.

The better you describe what you want, the better Claude's output will be. Here are some tips:

Be Specific

Vague

"Make me a website"

Specific

"Make me a personal portfolio website with a dark theme. Include sections for About Me, My Projects, and Contact Info. Use a modern, clean design."

Describe the Look and Feel

Vague

"Make it look nice"

Specific

"Use soft pastel colors (light pink and lavender), rounded corners on buttons, and a friendly sans-serif font"

Iterate and Refine

You don't have to get it perfect in one message! Build on what Claude makes:

You: Create a recipe card for chocolate chip cookies
Claude: [creates a recipe card]
You: I like it, but can you make the font bigger and add a photo placeholder at the top?
Claude: [updates the design]
You: Perfect! Now change the background color to a warm cream color.

Prompt Templates to Try

For websites:

Create a [type] website about [topic]. Use [colors/style]. Include sections for [section 1], [section 2], and [section 3]. Make it mobile-friendly.

For fixing something:

The [thing] isn't working right. When I [do this], it should [expected behavior], but instead it [actual behavior]. Can you fix it?

For changes:

In the [filename] file, change [specific thing] to [what you want instead].
Lesson 7

Your First Real Project: Build a Personal Website

+
Goal: Use Claude to build a complete personal website from scratch.

Let's put everything together and build something real! Follow along step by step.

1

Open PowerShell and create a new project folder:

mkdir my-website
cd my-website
2

Start Claude:

claude
3

Give Claude your first prompt. Customize this to make it about YOU:

Create a personal website for me. My name is [Your Name]. I enjoy [your hobbies]. Use a warm, friendly design with a light background. Include: 1) A hero section with a greeting, 2) An "About Me" section, 3) A "My Interests" section with cards for each hobby, 4) A footer with a fun quote. Make it modern and mobile-friendly.
4

Approve the files Claude creates (press y when asked).

5

Open your website in a browser:

explorer index.html
6

Now refine it! Go back to Claude and ask for changes:

  • "Change the color scheme to [colors you like]"
  • "Add a photo placeholder in the About Me section"
  • "Make the hobby cards have rounded corners and shadows"
  • "Add an animation when the page loads"
Keep going! The more you ask Claude to refine, the more personalized your site becomes. There's no limit to what you can ask for!

Lesson 9

Tips, Tricks & Troubleshooting

+
Goal: Quick reference for common issues and helpful tricks.

Common Issues & Fixes

"claude" is not recognized as a command

Fix: Close PowerShell and reopen it. If that doesn't work, reinstall Claude Code:

irm https://claude.ai/install.ps1 | iex

Then close and reopen PowerShell again.

Permission errors when installing

Fix: Open PowerShell as Administrator: press Windows, type "powershell", right-click it, and choose "Run as administrator."

Claude seems stuck or frozen

Fix: Press Escape to interrupt. If that doesn't work, press Ctrl + C twice to force quit, then start Claude again.

My website doesn't look right

Fix: Tell Claude what's wrong! Be specific: "The header text is too small" or "The colors don't match what I asked for." Claude can see and fix its own work.

Claude made a mistake or I want to undo

Fix: Just tell Claude: "Undo the last change" or "Go back to how it was before." You can also press Ctrl + Z in most text editors.

Power User Tips

  • You can drag and drop images into Claude's conversation to show it what you want something to look like.
  • If Claude's response is too long, type /compact to have it summarize the conversation.
  • Start each new project in its own folder to keep things organized.
  • If you want Claude to keep a certain style across sessions, create a file called CLAUDE.md in your project folder with your preferences.
  • You can copy-paste error messages directly to Claude and ask it to fix them.

Frequently Asked Questions

Is Claude Code free?

Claude Code requires an Anthropic account. There may be a free tier or trial available. Check the Anthropic website for current pricing. The Max plan ($100/month for Claude Pro with Claude Code included) or the API with pay-per-use are the most common options.

Do I need to know how to code?

Nope! That's the beauty of Claude Code. You describe what you want in plain English, and Claude writes the code. Over time, you'll naturally start to pick up some coding concepts, but it's not required.

Is it safe? Can Claude break my computer?

Claude always asks for your permission before making changes. It can't do anything without your approval. If you're unsure about something Claude wants to do, just say no! You can also ask Claude to explain what a command does before approving it.

What can I build with Claude Code?

Websites, simple apps, scripts to automate tasks, data analysis, games, and more. If you can describe it, Claude can probably help build it!

What if I mess something up?

Don't worry! You can always tell Claude to undo changes. Working in separate project folders keeps things isolated. And you can always delete a folder and start over — nothing is permanent.
